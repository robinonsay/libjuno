1. Introduction

1.1. Purpose

This Software Requirements Specification (SRS) defines the functional and non-functional requirements for LibJuno, a freestanding C11 embedded systems micro-framework.

This document follows the IEEE 830-1998 standard structure.

1.2. Scope

LibJuno provides data structures, memory management primitives, CRC algorithms, and other utilities for embedded systems development. It requires zero dynamic memory allocation and supports freestanding C11 environments.

1.3. Definitions and Acronyms

Term Definition

Freestanding

A C implementation that does not require a hosted standard library

DI

Dependency Injection — a pattern where dependencies are provided to a component rather than created internally

Vtable

A struct of function pointers enabling polymorphic dispatch

SRS

Software Requirements Specification

RTM

Requirements Traceability Matrix

1.4. References

  • IEEE 830-1998: Recommended Practice for Software Requirements Specifications

  • LibJuno README and documentation

2. Overall Description

2.1. Product Perspective

LibJuno is a standalone C11 library designed to be used as a dependency in embedded systems projects. It makes no assumptions about the target platform and uses dependency injection to remain adaptable.

2.2. Design Constraints

  • No dynamic memory allocation (malloc, calloc, realloc, free)

  • All memory is caller-owned and injected

  • Must compile as freestanding C11 (-nostdlib -ffreestanding)

  • All warnings treated as errors (-Werror)

2.3. Assumptions and Dependencies

  • A C11-compliant compiler is available

  • CMake ≥ 3.10 is available for building

  • Unity test framework is available for testing (build-time only)
