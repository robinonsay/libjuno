= Math Module

== Purpose

The Math module (`juno_math.h`, `juno_vec.h`, `juno_vec_types.h`, `juno_math_types.h`) provides generic mathematical operations, vector algebra, and quaternion types for embedded applications.
It uses macro-based generics and compile-time type selection for freestanding-compatible math.

== Data Structures

=== Scalar Types

`JUNO_MATH_F32_T` (`float`):: Single-precision floating-point.
`JUNO_MATH_F64_T` (`double`):: Double-precision floating-point.

=== Vector Types

`JUNO_MATH_VEC2_F32_T`, `JUNO_MATH_VEC2_F64_T`:: 2D vectors `{x, y}`.
`JUNO_MATH_VEC3_F32_T`, `JUNO_MATH_VEC3_F64_T`:: 3D vectors `{x, y, z}`.
`JUNO_MATH_VEC4_F32_T`, `JUNO_MATH_VEC4_F64_T`:: 4D vectors `{x, y, z, w}`.

=== Quaternion Types

`JUNO_MATH_QUAT_F32_T`, `JUNO_MATH_QUAT_F64_T`:: Quaternions `{x, y, z, w}`.

=== Result Types

`JUNO_MATH_F32_RESULT_T`, `JUNO_MATH_F64_RESULT_T`:: Scalar results.
`JUNO_MATH_VEC*_RESULT_T`:: Vector results for each dimension and precision.

== Interface Design

=== Generic Math Macros

`JunoMath_Abs(a)`:: Absolute value.
`JunoMath_Max(a, b)`:: Maximum of two values.
`JunoMath_Min(a, b)`:: Minimum of two values.
`JunoMath_Clamp(x, lo, hi)`:: Clamp value to range.

=== Vector Operations

`JunoMath_VecAdd(tVecA, tVecB)`:: Component-wise vector addition.
`JunoMath_VecSub(tVecA, tVecB)`:: Component-wise vector subtraction.
`JunoMath_VecScale(tVec, fScalar)`:: Scalar multiplication.
`JunoMath_VecDot(tVecA, tVecB)`:: Dot product.
`JunoMath_VecCross(tVecA, tVecB)`:: Cross product (3D only).
`JunoMath_VecNorm(tVec)`:: Euclidean norm (magnitude).
`JunoMath_VecNormalize(tVec)`:: Unit vector normalization.

Operations are implemented as `_Generic`-dispatched inline functions selecting the appropriate precision (F32 or F64) and dimension (Vec2, Vec3, Vec4) variant at compile time.

== Algorithm Descriptions

=== Dot Product

Sum of component-wise products: $\vec{a} \cdot \vec{b} = \sum_{i} a_i \cdot b_i$.

=== Cross Product (3D)

$\vec{a} \times \vec{b} = (a_y b_z - a_z b_y,\ a_z b_x - a_x b_z,\ a_x b_y - a_y b_x)$

=== Norm

$\|\vec{v}\| = \sqrt{\sum_{i} v_i^2}$

Uses `sqrtf` (F32) or `sqrt` (F64).

=== Normalize

$\hat{v} = \vec{v} / \|\vec{v}\|$

Returns error if norm is zero.

All operations are O(d) where d is the vector dimension (2, 3, or 4).

== Error Handling

* Division by zero in normalization: returns `JUNO_STATUS_ERR` with a zero vector.
* All vector operations return Result types; callers use `JUNO_ASSERT_OK` to validate.

== Design Rationale

The math module uses C11 `_Generic` for compile-time type dispatch because math operations are purely value-based — there is no runtime polymorphism or state injection needed.
This is the one place where `_Generic` is appropriate in LibJuno: it provides type safety and ergonomic dispatch without needing vtables.
The macro-based generic approach keeps the API consistent across precisions and dimensions.
