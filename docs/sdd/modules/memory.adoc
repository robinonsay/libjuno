= Memory Module

== Purpose

The Memory module (`memory_api.h`, `memory_block.h`) provides the abstract memory allocation interface and its fixed-size block allocator implementation.
It enables deterministic, bounded-capacity memory management on caller-supplied backing buffers without dynamic allocation.

== Data Structures

=== Abstract Allocator Root (`JUNO_MEMORY_ALLOC_ROOT_T`)

[source,c]
----
struct JUNO_MEMORY_ALLOC_ROOT_TAG JUNO_MODULE_ROOT(JUNO_MEMORY_ALLOC_API_T,
    const JUNO_POINTER_API_T *ptPointerApi;  // Dependent pointer operations
);
----

Follows the module root pattern with an additional dependency on the pointer API.

=== Allocator API (`JUNO_MEMORY_ALLOC_API_T`)

[cols="1,3"]
|===
| Function | Description

| `Get(ptMem, zSize)` | Allocate a region of at least `zSize` bytes. Returns `JUNO_RESULT_POINTER_T`.
| `Update(ptMem, ptMemory, zNewSize)` | Adjust an existing allocation's descriptor in-place.
| `Put(ptMem, pvAddr)` | Free a previously allocated region. Clears descriptor on success.
|===

=== Block Allocator Derivation (`JUNO_MEMORY_ALLOC_BLOCK_T`)

[source,c]
----
struct JUNO_MEMORY_ALLOC_BLOCK_TAG JUNO_MODULE_DERIVE(JUNO_MEMORY_ALLOC_ROOT_T,
    uint8_t *pvMemory;                        // Backing memory region
    JUNO_MEMORY_BLOCK_METADATA_T *ptMetadata; // Per-block free-list metadata
    size_t zTypeSize;                         // Element size per block
    size_t zAlignment;                        // Alignment per block
    size_t zLength;                           // Total blocks in pool
    size_t zUsed;                             // Blocks currently allocated
    size_t zFreed;                            // Entries on free stack
);
----

=== Block Metadata (`JUNO_MEMORY_BLOCK_METADATA_T`)

Each entry stores the address of a freed block; the free list is a stack in `ptMetadata[0..zFreed)`.

== Interface Design

`JunoMemory_BlockInit(...)`::
Initializes a block allocator over caller-supplied backing memory and metadata arrays.
Validates all parameters (nulls, overflow of `zTypeSize * zLength`, base alignment).
Wires the vtable into the root. Counters `zUsed` and `zFreed` start at 0.

`JunoMemory_BlockGetT(ptBlkRoot, type)`::
Convenience macro that calls `Get` with `sizeof(type)`.

`JunoMemory_BlockPutT(ptBlkRoot, pPtr)`::
Convenience macro that calls `Put` on a pointer descriptor.

`JunoMemory_AllocApiVerify(ptAllocApi)`::
Inline verification that all three API functions (Get, Put, Update) are non-NULL.

`JunoMemory_AllocVerify(ptAlloc)`::
Inline verification of the allocator root, its API, and its dependent pointer API.

=== Storage Declaration Macros

`JUNO_MEMORY_BLOCK(name, type, length)`::
Declares a static backing array of `type[length]`.

`JUNO_MEMORY_BLOCK_METADATA(name, length)`::
Declares a static metadata array of `JUNO_MEMORY_BLOCK_METADATA_T[length]`.

== Algorithm Descriptions

=== Block Get (Allocation)

. If a freed block exists (`zFreed > 0`): pop from free stack (LIFO), decrement `zFreed`.
. Otherwise: carve a new block from offset `zUsed * zTypeSize` if `zUsed < zLength`; increment `zUsed`.
. Reset the block via the pointer API before returning.
. On Reset failure, roll back and report the error.

*Complexity*: O(1).

=== Block Put (Deallocation)

. Verify the address belongs to the pool and is properly aligned.
. Reject double-frees.
. If the block is the most recently allocated (last in sequence): decrement `zUsed` directly.
. Otherwise: push the address onto the free stack; increment `zFreed`.
. Clear the caller's descriptor (`pvAddr = NULL`, sizes zeroed).

*Complexity*: O(1).

=== Block Update

. Adjust the descriptor's `zSize` in-place, up to `zTypeSize`.
. Reject requests exceeding `zTypeSize`. Memory is never moved.

*Complexity*: O(1).

== Error Handling

* `Get`: returns `JUNO_STATUS_MEMALLOC_ERROR` when the pool is full or size exceeds element size. Returns `JUNO_STATUS_NULLPTR_ERROR` for zero-size requests.
* `Put`: returns `JUNO_STATUS_MEMFREE_ERROR` for unaligned, out-of-range, or double-freed addresses. Restores the descriptor on failure.
* `Update`: returns `JUNO_STATUS_MEMALLOC_ERROR` if the new size exceeds element capacity.
* All operations call `Verify` at entry.

== Design Rationale

The abstract `JUNO_MEMORY_ALLOC_API_T` separates the allocation interface from the block allocator implementation, allowing future allocator strategies (slab, buddy) to be injected through the same root type.
The block allocator provides O(1) deterministic operations suitable for hard real-time systems with fixed-capacity pools.
